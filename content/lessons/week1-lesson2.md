# ビジネスアイデアの検証とMVP設計

## リーンスタートアップとは

エリック・リースが提唱した起業手法で、**最小限の投資で仮説を検証し、素早く学びを得る**アプローチです。

Vibe Codingとの相性が抜群です。なぜなら：
- MVPを**数日〜1週間**で構築できる
- ユーザーフィードバックに基づき**即座に改善**できる
- 失敗のコストが極めて低い

## ステップ1：ペルソナ設定

あなたのプロダクトを使う「理想的なユーザー」を具体的に描きます。

**テンプレート：**
- 名前（仮名でOK）
- 年齢・性別・職業
- 日常的な課題やフラストレーション
- 理想の状態
- 情報収集の方法

## ステップ2：課題-ソリューションフィット

以下の質問に答えてみましょう：

1. ユーザーが抱える**最大の課題**は何か？
2. その課題は現在どのように解決されているか？
3. 既存の解決策の**不満点**は何か？
4. あなたのソリューションはなぜ**より良い**のか？

## ステップ3：MVP機能要件の定義

### 「Must Have」と「Nice to Have」を分ける

MVP で**必ず必要な機能**だけをリストアップします。

**判断基準：**
- この機能がないとユーザーの課題は解決できない？ → Must Have
- あったら便利だが、なくても価値は提供できる？ → Nice to Have

### ユーザーストーリー形式で書く

```
「[ユーザー]として、[目的]のために、[機能]がほしい」
```

例：
- 「店長として、売上を把握するために、日別売上グラフが見たい」
- 「顧客として、予約を取るために、空き状況カレンダーを確認したい」

## ステップ4：技術スタック選定

| ツール | 向いている用途 | 難易度 |
|--------|---------------|--------|
| Replit | Webアプリ全般、初心者向け | 低 |
| Cursor | 本格的な開発、カスタマイズ重視 | 中 |
| v0.dev | UIデザイン・フロント特化 | 低 |
| Lovable | ランディングページ、簡単なアプリ | 低 |

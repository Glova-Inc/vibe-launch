# 開発ツールのセットアップと最初の一歩

## 環境構築

### Replit を使う場合

1. [replit.com](https://replit.com) にアクセス
2. Google / GitHub アカウントでサインアップ
3. 「Create Repl」をクリック
4. テンプレートは「Replit Agent」を選択

### Cursor を使う場合

1. [cursor.com](https://cursor.com) にアクセス
2. アプリをダウンロード & インストール
3. アカウント作成（Proプラン推奨）
4. 新しいプロジェクトフォルダを作成

## 最初のプロンプトの書き方

### 良いプロンプトの構造

```
【コンテキスト】何を作りたいか
【対象ユーザー】誰が使うか
【主要機能】必要な機能リスト
【技術要件】使いたい技術（任意）
【デザイン】見た目のイメージ
```

### 実践例

```
飲食店向けの予約管理アプリを作ってください。

対象ユーザー：
- 小規模レストランのオーナー（管理画面）
- 一般のお客様（予約画面）

必要な機能：
1. お客様が日時・人数を指定して予約できる
2. オーナーが予約一覧を確認・管理できる
3. 予約確認メールの自動送信

デザイン：
- シンプルで清潔感のあるデザイン
- モバイル対応
- 日本語UI
```

## プロンプトのコツ

1. **具体的に書く**: 「いい感じに」ではなく「青を基調とした清潔感のあるデザイン」
2. **段階的に進める**: 一度に全部作ろうとせず、まずコア機能から
3. **例を示す**: 参考サイトのURLを共有するのも効果的
4. **フィードバックは明確に**: 「ヘッダーの文字を大きくして、ボタンの色を緑に変更して」

## 今日のゴール

Week 1で定義したMVPのコア機能を、まず1つだけ動く状態にしましょう。完璧を目指す必要はありません。「動く」ことが最優先です。

# 品質向上：テストとセキュリティの基本

## なぜ品質が大切なのか

Vibe Codingで素早くプロダクトを作れる反面、品質面での注意が必要です。

### 「6ヶ月の壁」を避ける

業界では、AI生成コードの品質管理を怠ると約6ヶ月でメンテナンスが困難になると言われています。今のうちから基本を押さえましょう。

## セキュリティチェックリスト

### 必ず確認すべき項目

- [ ] APIキーやパスワードがコードに直書きされていないか
- [ ] ユーザー入力のバリデーション（検証）はされているか
- [ ] HTTPS（暗号化通信）は有効か
- [ ] 不要な情報が外部に公開されていないか

### AIに依頼する例

```
以下のセキュリティ観点でコードを見直してください：
1. ハードコードされた秘密情報がないか
2. SQL インジェクション対策は十分か
3. XSS（クロスサイトスクリプティング）対策は行われているか
4. 認証・認可は適切に実装されているか
```

## ユーザーテストの進め方

### テストの手順

1. **テスターを3〜5人集める**（友人、同僚、家族でOK）
2. **タスクを設定する**（例：「予約を完了してください」）
3. **観察する**（口を出さず、操作を見守る）
4. **フィードバックを記録する**
5. **優先度をつけて修正する**

### よくある発見

- ボタンの位置がわかりにくい
- エラーメッセージが不親切
- ページの読み込みが遅い
- モバイルで表示が崩れる

## AIで品質を改善するプロンプト例

```
このアプリのUI/UXを改善してください：
- ボタンのクリック可能エリアを大きくする
- エラーメッセージを日本語でわかりやすく表示する
- ローディング中はスピナーを表示する
- フォーム送信後に成功メッセージを表示する
```

## まとめ

品質向上は「後から」ではなく「今から」が鉄則。AIの力を借りれば、セキュリティチェックもテスト改善も効率的に進められます。
